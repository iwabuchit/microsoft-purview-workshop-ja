    Module 02B - データ資産の登録とスキャン(Azure SQL Database)

    はじめに

    📢 導入

    データを検出し把握するために Microsoft Purview にデータ資産を登録するには、データ資産をソースとして登録してスキャン機能を利用できるようにする必要があります。スキャンにより Microsoft Purview は、完全修飾名、スキーマ、データ型などの技術的なメタデータを抽出し、元になるデータのサンプルを解析して分類を適用できます。このモジュールでは、データ ソースの登録とスキャン方法について説明します。

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Azure SQL Database (Module 00 を参照)
    • Microsoft Purview アカウント (Module 01 を参照)

    🎯 ゴール

    • Azure SQL Databaseを登録し、Azure Key Vaultに保存されているSQL認証の資格情報を使用してスキャンを実施する

    アクセスポリシーの設定 #1

    Note

    Azure Key Vault は、シークレットの安全なストアを提供するクラウド サービスです。Azure Key Vault を使用すると、キー、パスワード、証明書、その他のシークレットを安全に格納できます。詳細については、「Azure Key Vault について」を参照してください。

    1. Azure Key Vaultリソースに移動し [アクセス ポリシー] をクリックします。

    2. [＋ 作成] をクリックします。

    3. [シークレットのアクセス許可] で [すべて選択] を選択し [次へ] クリックします。

    4. 自分の [アカウント名] を検索し、検索結果から自分のアカウントを選択し [確認および作成] をクリックします。

    5. 選択した内容を確認し [作成] をクリックします。

    アクセスポリシーの設定 #2

    Note

    この手順では、キー コンテナーからシークレットを取得するために必要なアクセスを Microsoft Purview に提供する 2 番目のアクセス ポリシーを作成します。

    1. [アクセス ポリシー] をクリックします。

    2. [＋作成] をクリックします。

    3. [シークレットのアクセス許可] で [取得] と [一覧] を選択し [次へ] クリックします。

    4. Microsoft Purviewアカウントの名前（例: pvlab-{randomId}-pv）を検索し、アカウントを選択して [確認および作成] をクリックします。

    5. 選択内容を確認し [作成] をクリックします。

    シークレットの作成

    1. [シークレット] に移動し [＋ 生成/インポート] をクリックします。

    2. [名前] と [値] に次の値を入力して [作成] をクリックします。

    3. 名前: 

      sql-secret

    4. 値: 

      sqlPassword!

    資格情報をMicrosoft Purviewに追加する

    1. [Purview ガバナンス ポータル] から [管理] ＞ [資格情報] に移動し [Key Vault接続の管理] へ移動します。

    2. [新規作成] をクリックします。

    3. Key Vaultの接続名をmyKeyVaultと設定、適切なサブスクリプションとKey Vault名を選択して [作成] をクリックします。

    4. [閉じる] をクリックします。

      Note

      アクセス権の付与の確認画面が表示された場合は [確認] をクリックしてください。

    5. [資格情報] で [＋ 新規] をクリックします。

    6. 認証方法をSQL 認証に設定しKey Vault接続をmyKeyVaultに設定します。それぞれのフィールドに以下の値を入力して [作成] をクリックします。

      • 名前: 
        credential-SQL
      • ユーザー名: 
        sqladmin
      • シークレット名: 
        sql-secret

    ソースの登録（Azure SQL Database）

    1. [データ マップ] ＞ [データ ソース] に移動し [登録] をクリックします。

    2. [Azure] タブへ移動し [Azure SQL Database] を選択し [続行] をクリックします。

    3. [Azure サブスクリプション] 、[サーバー名]、[コレクション] の順に選択し [登録] をクリックします。

    登録した資格情報を利用したスキャン

    1. [データマップ] ＞ [データ ソース] に移動し [Azure SQL Database] タイル内で [新しいスキャン] アイコンをクリックします。

    2. [データベース] と [資格情報] を選択し [接続のテスト] をクリックします。接続が成功したら [続行] をクリックします。

      Warning

      ServerlessタイプのSQL Databaseを選択しているため、DB作成から時間が経過しているとDBが停止しており、テストに失敗する場合があります。その場合は、再度接続テストをお試しください。

    3. [続行] をクリックします。

    4. [続行] をクリックします。

    5. [一度だけ] を選択し [続行] をクリックします。

    6. [保存して実行] をクリックします。

    7. スキャンの進行状況を監視するには [詳細の表示] をクリックします。

    8. [最新の情報に更新] をクリックしてスキャンの状態を定期的に更新します。

    アセットの表示

    1. [データ カタログ] に移動し検索ボックスに *（アスタリスク）を入力しEnterを押してワイルドカード検索を実行します。
