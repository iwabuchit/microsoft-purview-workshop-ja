    Module 03 - アセットの検索と参照

    はじめに

    📢 導入

    ソースが登録されスキャンされると、データカタログに実際のオブジェクト（Azure SQL Database のテーブルや Power BI のレポートなど）を表すアセットが追加されます。

    このモジュールでは、テーブルやカラムなどのデータ属性に定義や分類を追加して、テクニカル メタデータを編集する方法やアセットの属性を担当者にリンクさせることで所有者を割り当てる方法、分類を使用してデータをマークする方法を学びます。

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Microsoft Purview アカウント (Module 01 を参照)
    • Microsoft Purview カタログといくつかのアセット（Module 02a Module 02b を参照）

    🎯 ゴール

    • キーワードでカタログを検索する
    • カタログをソース別に閲覧する
    • 登録済みのアセットを更新する
    • アセットの一括編集を行う

    カタログの検索

    1. Purview ガバナンス ポータル を開き[データカタログ]画面からアスタリスク*を検索バーに入力しEnterを押します。

    2. [分類]で検索結果を[Country/Region]でフィルターしアセット名QueriesByStateをクリックして詳細を表示します。

      Note

      Purview ガバナンス ポータルは既定のユーザーエクスペリエンスとして提供されていますが、顧客やパートナーはMicrosoft Purview REST APIを使用して基盤となるプラットフォームとインターフェースできます。プラットフォームがオープンであるため、Microsoft Purviewとカスタムユーザーインターフェースを統合したり、プログラムで読み取り/書き込み操作を実行したりすることが可能です。

    アセットの更新

    1. [編集]をクリックしてアセットの詳細を変更します。

    2. 以下のサンプルテキストを[資産の説明]に貼り付けます。

      This dataset was curated from the Bing search logs (desktop users only) over the period of Jan 1st, 2020 – (Current Month - 1). Only searches that were issued many times by multiple users were included. The dataset includes queries from all over the world that had an intent related to the Coronavirus or Covid-19. In some cases this intent is explicit in the query itself (e.g., “Coronavirus updates Seattle”), in other cases it is implicit , e.g. “Shelter in place”

    3. ドロップダウン メニューから[分類]にWorld Citiesを割り当てます。

    4. [スキーマ]タブに移動しサンプルテキストで各列の[資産の説明]を更新します。

      • Date: 
        Date on which the query was issued.
      • Query 
        The actual search query issued by user(s).
      • IsImplicitIntent 
        True if query did not mention covid or coronavirus or sarsncov2 (e.g, “Shelter in place”). False otherwise.
      • State 
        State from where the query was issued.
      • Country 
        Country from where the query was issued.
      • PopularityScore 
        Value between 1 and 100 inclusive. 1 indicates least popular query on the day/State/Country with Coronavirus intent, and 100 indicates the most popular query for the same geography on the same day.

    5. [連絡先]タブに移動し、組織内のユーザを[エキスパート]と[所有者]に設定し[保存]クリックします。

      Note

      アセットは、次の 2 種類の連絡先に関連付けることができます。
      - エキスパート: 多くの場合、ビジネスプロセスまたは対象分野の専門家です。
      - 所有者: 多くの場合、特定のデータ領域に対するガバナンスまたはビジネス プロセスを定義する上級管理職またはビジネス領域の所有者です。
      連絡先としてタグ付けされているアセットの場合、これらはホーム画面(データカタログ)の[マイ アイテム]の下に表示されます。

    6. 同じパス内の他のアセットを表示するには[関連]タブに移動します。

    アセットの参照

    検索エクスペリエンスはキーワード ベースの検出に最適ですが、Microsoft Purview ガバナンス ポータルでは、資産を参照する別の方法 (コレクション別またはソースの種類別) を使用できます。

    1. [データ カタログ]画面から[参照]をクリックします。

    2. [ソースの種類別]タブに切り替えてAzure Data Lake Storage Gen2を選択します。

    3. アカウントpvlab{randomId}adlsを選択します。

    4. コンテナーrawを選択します。

    一括編集

    Microsoft Purview を使用すると、属性のサブセット (エキスパート、所有者、用語、分類) に対して特定の操作 (追加/置換/削除) を Microsoft Purview ガバナンス ポータル内で直接一括で実行できます。

    1. [データカタログ]画面からアスタリスク*を検索バーに入力しEnterを押します。

    2. フィルター内の資産の種類から[データ]を選択します。

    3. [ファイル]と[表]でフィルターをして次の5つの項目を選択し[選択項目の表示]をクリックします。

      • QueriesByState
      • Address
      • Customer
      • QueriesByCountry
      • CustomerAddress

    4. [一括編集]をクリックします。

    5. [属性]を[所有者]に、[操作]を[追加]に設定し組織内の1人以上のユーザを選択して[適用]をクリックします。

    6. [すべての選択を解除して閉じる]をクリックします。
