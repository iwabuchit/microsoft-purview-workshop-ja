    Module 01 - Purview Accountの作成

    はじめに

    📢 導入

    Microsoft Purviewは、大規模なデータの管理とガバナンスを支援する統合データガバナンスソリューションです。メタデータのスキャン、収集、整理、共有のためのさまざまなサービスを統合しています。データ資産を発見しやすいようにカタログを使用しますが、自動分類、共有、アクセス管理などの機能も提供しています。

    Microsoft Purview を導入する際には、まず Microsoft Purview アカウントをプロビジョニングする必要があります。次に、データ資産をまとめて管理・グループ化するためのコレクションを定義する必要があります。コレクションは、データソースなどのメタデータを管理するための論理的なコンテナです。コレクションを作成して階層化する際には、セキュリティ要件、ガバナンス構造、利用者のニーズなど、さまざまなことを考慮する必要があります。例えば、中央集権的なデータ管理スタイルと、ドメイン指向的なデータ管理スタイルでは、コレクションの構造が異なります。ニーズに応じて、データ管理に異なるトポロジーを使用することができます。そのためのベストプラクティスは、こちらでご覧いただけます。

    🤔 前提条件

    • 有効なサブスクリプションを持つAzureアカウント
    • Azureサブスクリプションでリソースを作成する権限
    • サブスクリプションには、次のリソース プロバイダーが登録されている必要があります。
    • Microsoft.Purview
    • Microsoft.Storage

    • Microsoft.EventHub

      Note

      Azure ポータルからリソース プロバイダを登録する方法については、こちらを参照してください。

    🎯 ゴール

    • Azureポータルを使用して、Microsoft Purviewのアカウントを作成する
    • Microsoft Purviewのデータプレーンにアクセスできるユーザーを追加する

    Purviewアカウントの作成

    1. Azureポータルにサインインしホーム画面から [リソースの作成] をクリックします。

    2. マーケットプレイスでMicrosoft Purviewを検索し [作成] をクリックします。

    3. [基本] タブで必要な情報を入力します。

      Note

      以下は、説明のための例示でありご自身の環境に合わせた値を指定するようにしてください。ラボのテンプレートを使用して他の Azure リソースを事前に展開した場合 randomId で作成されます。以下の例のように、Microsoft Purview アカウント名には同じ randomId を使用することをお勧めします。

      • サブスクリプション 
        ご利用のサブスクリプション
      • リソースグループ 
        purview-workshop-rg
      • Purviewアカウント名 
        pvlab-{randomId}-pv
      • 場所 東日本など有効な場所
      • マネージド リソース グループ名 
        managed-rg-pvlab-{randomId}-pv

    4. [ネットワーク] タブで [すべてのネットワーク] を選択します。

    5. [確認および作成] タブでリボンのメッセージが [検証に成功しました] と表示されたら内容を確認し [作成] をクリックします。

    6. デプロイが完了するまで数分待ちます。完了したら [リソースに移動] をクリックします。

    データ プレーンへのアクセス許可設定

    デフォルトでは、Microsoft Purview アカウントリソースの作成に使用された ID は、Microsoft Purview ガバナンスポータルへのフルアクセスを持ちます。以下の手順では、Azure Active Directory 内の追加のユーザーにアクセスを提供する方法について説明します。

    1. AzureポータルからPurviewアカウントへ移動し [Purview ガバナンス ポータル を開く] タイルをクリックします。

    2. 左側の [データ マップ] に移動します。

    3. [コレクション] を選択します。

    4. [ロールの割当て] を選択し[データ キュレーター] 右側にある [追加] アイコンをクリックします。

    5. Azure Active Directory内の別のユーザを検索し、アカウントを選択して [OK] をクリックします。
