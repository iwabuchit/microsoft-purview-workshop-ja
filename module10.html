Module 10 - セルフホステッド統合ランタイムの構成

はじめに

📢 導入

オンプレミスのデータソースからMicrosoft Purviewにアセットを投入するためには、セルフホステッド統合ランタイムを使用して、データの発見と探索のためのゲートウェイを作成する必要があります。このモジュールでは、セルフホステッド統合ランタイムのインストール、オンプレミスのSQLサーバーの登録、データソースのスキャンの方法を説明します。

🤔 前提条件

  • アクティブなサブスクリプションを持つAzureアカウント
  • Microsoft Purview アカウント (Module 01 を参照)

🎯 ゴール

  • システム・デフォルト用語テンプレートを使用して、用語集に用語を作成する
  • カスタム用語テンプレートを使用して用語集に用語を作成する
  • CSVファイルを使用して用語を用語集に一括インポートする
  • 用語集からCSVファイルに用語を一括エクスポートする
  • データカタログのアセットに用語を割り当てる
  • 既存の用語を関連用語と連絡先で更新する

SQL 仮想マシンへの接続

  1. セルフホステッド統合ランタイムのインストールの為、SQL Server 仮想マシンにログインします Azure ポータルの [Virtual Machines]より SQL 仮想マシンを選択します。(e.g. pvlab{randomId]sqlvm) [接続] > [RDP] > [RDP ファイルのダウンロード] と進み、RDP ファイルをローカルにダウンロードします。

  2. SQL 仮想マシンへ接続する為の、ユーザー名とパスワード（module 00 で作成）を取得します。 Azure ポータルの[リソース グループ]より今回作成した「リソース グループ名」を選択します。 続いて、[デプロイ] > [SQLVMDeployment] > [出力] と進み、ユーザー名とパスワードをメモします。

  3. 先ほどダウンロードした RDP ファイルを開き、[接続]をクリックします。

  4. [その他] > [別のアカウントを使用する]より メールアドレス：先ほど取得したユーザー パスワード：先ほど取得したパスワード を入力します。警告のポップアップが表示されますが、[はい]をクリックします。

セルフホステッド統合ランタイムのインストール

  1. 今回はハンズオン時の一時利用の VM なので、IE のセキュリティ要件を一時的に緩和して手順を簡素化します。本番における同様の対応を推奨するものではありません。

  2. ブラウザを開き、https://www.microsoft.com/en-us/download/confirmation.aspx?id=39717 より最も新しい統合ランタイムをダウンロードし、そのまま実行します。

  3. 次の手順にしたがって、インストールを実施します。

    !!! Note 統合ランタイムは、異なるネットワーク環境でデータ統合機能を提供し、これらのアクティビティがデータ ストアに可能な限り近い地域で実行されるようにするために使用されるセキュアなコンピュートインフラストラクチャです。 セルフホステッド統合ランタイムは、オンプレミスのマシンまたは仮想ネットワーク内の仮想マシンにインストールすることが可能です。

  4. もし、[Microsoft Integration Runtime 構成マネージャー]が自動的に起動してこない場合は、Windows の[Start] > [Microsoft Integration Runtime]をクリックして起動できることを確認します。起動できることが確認できたら、先に Microsoft Purview 側での設定を行います。

Microsoft Purview における認証設定

  1. Microsoft Purview ガバナンス ポータル から、[データ マップ] > [統合ランタイム] > [新規] と進みます。

    !!! Note Microsoft Purview の統合ランタイムは、Azure Synapse Analytics や Azure Data Factory の統合ランタイムと同一マシン上で稼働させることはできません。

  2. [セルフホステッド]を選択し、[続行]をクリックします。

  3. 名前：任意の名前を入力（入力必須、今回はハンズオンなのでデフォルトの名前で OK） 説明：ブランクでも可 を入力し、[作成]をクリックします。

  4. 表示された Key1 の認証キーをコピーしてメモ帳等に貼り付けておき、[閉じる]をクリックします。

  5. 仮想マシン上の Microsoft Integration Runtime 構成マネージャーにコピーした認証キーを貼り付け、[登録]をクリックします。

  6. [Integration Runtime (Self-hosted) ノード名]の右側に緑色のチェックがついたことを確認し、[完了]をクリックします。

  7. [構成マネージャーの起動]をクリックし、”セルフホステッド ノードはクラウド サービスに接続されています“と表示されたことを確認します。

  8. Microsoft Purview ガバナンス ポータル から、[データ マップ] > [統合ランタイム] でセルフホステッドの状態が実行中になっていれば認証の設定は終了です。

    !!! Note Purview Integration Runtimeは、Azure Data Lake、Azure SQL Database、Azure Cosmos DBなど、プライベートエンドポイントの後ろにある各種の Azure クラウド サービスからメタデータ資産をスキャンして取り込むこともできます。
