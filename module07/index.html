    Module 07 - データ資産の分析情報

    はじめに

    📢 導入

    分析情報は、データソース管理者、ビジネスユーザー、データスチュワード、データオフィサー、セキュリティ管理者に特定のインサイトを提供することも目的としています。Microsoft Purviewには現在、以下のレポートがあります。

    正常性

    • データスチュワードシップ

    インベントリと所有権

    • アセット

    キュレーションとガバナンス

    • 用語集
    • 分類
    • 秘密度ラベル

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Microsoft Purview アカウント (Module 01 を参照)
    • スキャンのセットアップ完了（Module 02a Module 02b を参照）

    🎯 ゴール

    • レポートから収集できるさまざまな種類の分析情報を理解する

    データ スチュワードシップ インサイト

    1. Microsoft Purview ガバナンス ポータル を開きます。 次に、[データ資産の分析情報] > [データ スチュワードシップ] をクリックします。

      Warning

      Data Estate Insightsへ情報が反映されるまでスキャン後3～8時間ほど必要になる場合があります。
      データに関する Microsoft Purview のアセットの分析情報 - Microsoft Purview | Microsoft Docs

    2. データ スチュワードシップには、以下のようなメトリクスが表示されます。

      • キュレーションステータス別の資産数
      • 所有状況別の資産数
      • 月間アクティブユーザー数

    3. データ スチュワードシップには、さらに以下のデータの可視化表現があります。通常、これらのタイルでは、インタラクティブなフィルタリングと、「詳細を表示」をクリックすることで詳細へのドリルダウンが可能になっています。

      データ資産の正常性

      資産のキュレーション

      傾向とギャップの分析

    アセットから見た分析情報

    1. Microsoft Purview ガバナンス ポータル を開きます。 次に、[データ資産の分析情報] > [アセット] をクリックします。

    2. アセットには、以下のようなメトリクスが表示されます。

      • 資産数
      • 分類別保有資産数
      • 所有者別資産数
      • 新規資産数（過去30日間）
      • 削除資産数（過去30日間）

    3. アセットでは以下のグラフが表示されます。

      Note

      [ソースの種類別のデータ資産] グラフのクイック フィルターを使用し、[詳細の表示] をクリックして詳細を掘り下げると、特定の種類のデータを含むソースをすばやく簡単に識別できます (たとえば、 [分類カテゴリ] フィルターを Personal に設定します。)

      コレクションごとのデータ資産/ソース種類別のデータ資産

      ファイルベースのソースの種類

    用語集から見た分析情報

    1. Microsoft Purview ガバナンス ポータル を開きます。 次に、[データ資産の分析情報] > [用語集] をクリックします。

    2. 用語集には、以下のメトリックが含まれます。

      • 用語の合計
      • 資産と紐づかない承認済みの用語の合計
      • 資産と紐づく期限切れの用語の合計

    3. アセットでは以下のグラフが表示されます。

      アセット数ごとの上位条件

      用語は、定義、エキスパート、またはスチュワードが欠落している場合、不完全であると見なされます。1 つの用語にこれらの項目が複数欠落している場合は、「複数のアイテムが見つかりません」と表示されます。

    分類結果から見た分析情報

    1. Microsoft Purview ガバナンス ポータル を開きます。 次に、[データ資産の分析情報] > [分類] をクリックします。

    2. 分類には、以下のメトリックが含まれます。

      • 分類された資産の総数
      • 分類されたファイルの総数
      • 分類されたテーブルの総数
      • 一意の分類の総数
      • 分類されたソースの総数

    3. 分類では以下のグラフが表示されます。

      分類されたデータを含む上位のデータ ソース (過去 30 日間)

      上位の分類カテゴリ

      上位の分類

    秘密度ラベルから見た分析情報

    1. Microsoft Purview ガバナンス ポータル を開きます。 次に、[データ資産の分析情報] > [秘密度ラベル] をクリックします。

      Note

      秘密度ラベルを適用する Microsoft Purview の機能は、Microsoft Purview コンプライアンス ポータル内で制御されます。注: 秘密度ラベルを自動的に適用する利点を提供するアクティブな Microsoft 365 ライセンスが必要です。詳細については、「Microsoft Purview データ マップ内のデータに秘密度ラベルを自動的に適用する方法」を参照してください。

    2. 秘密度ラベルには、以下のメトリックが含まれます。

      • ラベル付き資産の総数
      • ラベル付きファイルの総数
      • ラベル付きテーブルの総数
      • 一意のラベルの合計数
      • ラベル付きソースの総数

    3. 秘密度ラベルでは以下のグラフが表示されます。

      ラベル付きデータを含む上位のデータ ソース (過去 30 日間)

      データ ソース全体に適用される上位のラベル

      上位のラベル
