    Module 06 - データ系列の可視化

    はじめに

    📢 導入

    Microsoft Purviewのプラットフォーム機能の1つに、データプロセスで作成されたデータセット間のリネージを表示する機能があります。Data Factory、Data Share、Power BIなどのシステムは、データが移動する際のリネージをキャプチャします。カスタムリネージレポートは、AtlasフックやREST APIでもサポートされています。 このモジュールでは、データリネージを追跡するためにAzure Data FactoryアカウントをMicrosoft Purviewアカウントに接続するために必要なことを順を追って説明します。

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Azure Data Lake Storage Gen2 アカウント (Module 00 を参照)
    • Azure Data Factory アカウント (Module 00 を参照)
    • Microsoft Purview アカウント (Module 01 を参照)

    🎯 ゴール

    • システム・デフォルト用語テンプレートを使用して、用語集に用語を作成する
    • Azure Data FactoryアカウントとMicrosoft Purviewアカウントを接続する
    • Azure Data Factoryのパイプラインをトリガーして実行し、リネージメタデータをPurviewにプッシュする

    Azure Data Factory 接続の作成

    1. Purview ガバナンス ポータル を開きます。 次に、[管理] > [Data Factory] > [新規]をクリックします。

      Note

      Data Factory 接続を表示/追加/削除するには、ルート コレクションに対するコレクション管理者ロールが割り当てられている必要があります。

    2. ドロップ ダウンメニューより、Azure Data Factory インスタンスを選択し、[OK]をクリックします。

      Note

      Microsoft Purview は、複数の Azure Data Factory に接続できますが、Azure Data Factory アカウントは、1 つの Microsoft Purview アカウントにしか接続できません。

    3. Data Factory 接続が正しく作られたことを確認します。

    4. [データ マップ] > [コレクション] > [YOUR_ROOT_COLLECTION] > [ロールの割り当て] データ キュレーターに Azure Data Factory のマネージド ID (e.g. pvlab-{ランダム}-adf)が追加されていることを確認します。

      Note

      Azure Data Factory の接続を作成すると、Azure Data Factory のマネージド ID がデータ キューレーター ロールに追加されます。 これにより、Azure Data Factory は、パイプラインの実行中にリネージを Microsoft Purview にプッシュすることが可能になります。

    Azure Data Factory を使ったデータのコピー

    1. Azure ポータル画面より、データ ファクトリに進み[Azure Data Factory Studio]を起動します。

    2. Azure Data Factory Studio を開き、[取り込み]をクリックします。

    3. [組み込みコピー タスク]を選択し[次へ]をクリックします。

    4. [ソースの種類]で[Azure Data Lake Storage Gen2]を選択し、[新しい接続]をクリックします。

    5. [Azure サブスクリプション]で現在ログインしているサブスクリプションを選択、[ストレージ アカウント名]に[e.g. pvlab{randomId}adls]を選択します。次に[テスト接続]が成功することを確認後[作成]をクリックします。

    6. ファイルまたはフォルダーの[参照]をクリックします。

    7. [raw/BingCoronavirusQuerySet/2020/]の順に辿り、[OK]をクリックします。

    8. 続いてフォルダーのパスを確認し、[次へ]をクリックします。 [列区切り記号]を[タブ]、[先頭行をヘッダーとして]にチェックして[データのプレビュー]をクリックし、 サンプル データを表示できたら、[次へ]をクリックします。

    9. ターゲットの種類：[Azure Data Lake Storage Gen2] 接続：[e.g. AzureDataLakeStorage1]（前の手順で作成した接続先）を選択した後に、フォルダーのパスで[参照]をクリックします。

    10. [raw]をクリックし、下図の状態にして、[OK]をクリックします。

    11. フォルダーのパスが以下のようになっていることを確認します。 ファイル名には[2020_merged.parquet]と入力し、コピー動作に[ファイルのマージ]を選択して、[次へ]をクリックします。

    12. ファイル形式が[Parquet 形式] になっていることを確認して、[次へ]をクリックします。

    13. そのまま[次へ]をクリックします。

    14. 概要画面にて[次へ]をクリックします。

    15. 配置が完了したら、[終了]をクリックします。

    16. モニター画面にて、パイプラインの実行が成功していることを確認します。

    データ系列(Lineage)の可視化

    1. Microsoft Purview ガバナンス ポータル のデータ カタログ画面にて、[アセットの参照]をクリックします。 [ソースの種類別]に切り替え、[Azure Data Factory]を選択します。

    2. Azure Data Factory インスタンスを選択、パイプラインの[CopyPipeline_xxx]から、アクティビティ[Copy_xxx]を選択します。

    3. [系列]タブをクリックします。

    4. Azure Data Factory から Microsoft Purview に自動的に送られたデータの系列情報が確認できます。 左側にはソースフォルダにある共通のスキーマを持つ 2 つのファイルのセット、中央にはコピー アクティビティ(プロセス)、右側には出力ファイルが配置されています。

      Note

      データセット： プロセスへの入力として提供されるデータセット（構造化または非構造化）。例えば、SQLテーブル、Azureブロブ、ファイル（.csvや.xmlなど）はすべてデータセットと見なされます。Purviewのリネージセクションでは、データセットは長方形のボックスで表されます。

      プロセス： データセットに対して実行されるアクティビティや変換をプロセスと呼ぶ。例えば、ADFコピーアクティビティ、データ共有スナップショットなどです。Purviewのリネージセクションでは、プロセスは丸みを帯びたボックスで表されます。
