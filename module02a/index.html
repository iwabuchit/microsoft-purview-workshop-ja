    Module 02A - データ資産の登録とスキャン(Azure Data Lake Storage Gen2)

    はじめに

    📢 導入

    データを検出し把握するために Microsoft Purview にデータ資産を登録するには、データ資産をソースとして登録してスキャン機能を利用できるようにする必要があります。スキャンにより Microsoft Purview は、完全修飾名、スキーマ、データ型などの技術的なメタデータを抽出し、元になるデータのサンプルを解析して分類を適用できます。このモジュールでは、データ ソースの登録とスキャン方法について説明します。

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Azure Data Lake Storage Gen2 アカウント (Module 00 を参照)
    • Microsoft Purview アカウント (Module 01 を参照)

    🔨 Tools

    • Azure Storage Explorer

    🎯 ゴール

    • コレクションの作成
    • Microsoft PurviewのマネージドIDを使用して、Azure Data Lake Storage Gen2アカウントを登録しスキャンを実施

    Microsoft PurviewのマネージドIDへのアクセス許可

    1. ADLS Gen2 pvlab{RandomID}adls アカウントへ移動し、ナビゲーションメニューから [アクセス制御(IAM)] を選択します。

    2. [ロールの割当の追加] をクリックします。

    3. [ストレージ BLOB データ閲覧者] を検索、選択し [次へ] をクリックします。

    4. [メンバー] タブへ移動し [アクセスの割り当て先] で [マネージド ID] を選択し [＋メンバーを選択する] をクリックします。 [マネージドID] ドロップダウン メニューから [Purview アカウント] を選択し作成したPurviewアカウント（pvlab-{randomId}-pv）を選び [選択] をクリックします。

    5. [レビューと割り当て] をクリックし、ロールの割り当てを実行します。

    6. [ロールの割り当て] タブに移動し、 [スコープ] を [このリソース] に設定しPurviewマネージドIDに [ストレージ BLOB データ閲覧者] の役割が付与されていることを確認します。

    ADLS Gen2へのデータ アップロード

    1. Azure ストレージ エクスプローラーをダウンロードしてインストールします。 Azureストレージ エクスプローラーを起動します。 [アカウントの管理]＞[表示] からAzureにサインインしアカウントを追加します。

    2. Bing コロナウィルス クエリセットのコピーをローカルマシンにダウンロードします。ダウンロードしたzipファイルを探し、ファイルを右クリックして [すべて展開...] を選択しコンテンツを解凍します。

    3. [展開] をクリックします。

    4. Azure ストレージ エクスプローラーを開き、サブスクリプションを展開して、ストレージアカウントを探します。 [Blob Containers] を右クリックし [BLOB コンテナーの作成] を選択しコンテナーを作成します。※ ここでは、作成するBLOBコンテナーの名前を「raw」とします。

    5. コンテナーを選択した状態で [アップロード] をクリックし [フォルダのアップロード] を選択します。

    6. 解凍したBingCoronavirusQuerySetフォルダーを選択し [フォルダーの選択] をクリックします。

    7. [アップロード] をクリックします。

    8. 転送が完了したことを確認します。

    コレクションの作成

    1. [データ マップ] ＞ [コレクション] に移動して [＋新しいコレクション] をクリックします。

    2. コレクションに [表示名] （Contoso）を指定し [作成] をクリックします。

    ソースの登録（ADLS Gen2）

    1. [データ マップ] ＞ [ソース] に移動し [登録] をクリックします。

    2. Azure Data Lake Storage Gen2を選択し [続行] をクリックします。

    3. [Azure サブスクリプション] 、 [ストレージ アカウント名] 、 [コレクション] の順に選択し [登録] をクリックします。

    マネージドIDを利用したスキャン

    1. [データマップ] ＞ [ソース] に移動し [Azure Data Lake Storage Gen2] タイル内で [新しいスキャン] アイコンをクリックします。

    2. [接続のテスト] をクリックしてMicrosoft Purview マネージドIDが適切なアクセス権を持っていることを確認します。接続に成功した場合は [続行] をクリックします。

    3. 階層を展開してスキャン スコープ内に含まれるアセットを確認し [続行] をクリックします。

    4. システム規定のスキャン ルール セットを選択し [続行] をクリックします。

    5. [一度だけ] を選択し [続行] をクリックします。

    6. [保存して続行] をクリックします。

    7. スキャン実行の進行状況を監視するには [詳細の表示] をクリックします。

    8. [最新の情報に更新] をクリックしてスキャンの状態を定期的に更新します。

      Info

      スキャンの完了まで約6分程度かかります。

    アセットの表示

    1. [データ カタログ] に移動し検索ボックスにアスタリスク（*）を入力し Enter を押してワイルドカード検索を実行します。

    2. 検索結果内にアセットのリストが表示されます。
