    Module 05 - 分類

    はじめに

    📢 導入

    Microsoft Purviewには、データ資産に存在する可能性のある典型的なデータ タイプを表す多数のデフォルト分類が用意されています（例：電子メールアドレス、クレジットカード番号、パスポート番号など）。また、デフォルトの分類がニーズに合わない場合は、カスタム分類を作成することができます。

    🤔 前提条件

    • アクティブなサブスクリプションを持つAzureアカウント
    • Azure Data Lake Storage Gen2 アカウント (Module 00 を参照)
    • Microsoft Purview アカウント (Module 01 を参照)

    🎯 ゴール

    • カスタム分類の作成
    • カスタム分類を資産に適用するスキャンの実行

    分類の作成

    1. Purview ガバナンス ポータルを開き[データマップ]＞[分類]に移動して[新規]をクリックします。

    2. 以下の値を[コピー]して適切なフィールドに貼り付け[OK]をクリックします。

      • 名前 
        Twitter Handle
      • 説明 
        TwitterのURLの末尾に表示されるユーザー名。

    3. [カスタム]タブに移動してカスタム分類が作成されたことを確認します。

    カスタム分類ルールの作成（正規表現）

    1. [データマップ]＞[分類ルール]に移動し[新規]をクリックします。

    2. 次の例に従って分類規則フィールドを設定し[続行]をクリックします。

      • 名前 
        twitter_handle
      • 説明 
        TwitterのURLの末尾に表示されるユーザー名。
      • 分類名 
        Twitter Handle
      • 状態 
        有効
      • 種類 
        Regular Expression

      Note

      分類ルールには2つのタイプがあります。正規表現は、実際のデータ名や列名に対してパターンマッチングを実行します。辞書ベースの分類ルールでは、CSVまたはTSVファイルを介してすべての可能な値のリストを提供できます。

    3. twitter_handles.csvをローカルマシンにダウンロードします。
      [参照]アイコンをクリックしダウンロードしたファイルを開きます。
      [Handle]列に関連付けられているデータ パターンを選択し[パターンに追加]をクリックします。

    4. [データ パターン]のプラス記号（+）を{5,15}で置き換えて変更します。

      Note

      プラス記号 (+) は、前の項目に一致する 1 つ以上の文字を示します。これにより、無制限の数の英数字が許可されるため、誤検知が発生する可能性があります。Twitter ハンドルは 5 文字以上 15 文字以下である必要があります。
      {5,15}を使用すると、前の項目が 5 回以上 15 回以下である場合にのみ条件に一致します。

    5. [列パターン]を変更することもできますが、この例ではデータ パターンにみを設定します。
      [列パターン]の入力をクリアし[作成]をクリックします。

    スキャン ルール セットの作成

    1. [データマップ]＞[スキャン ルール セット]に移動し[新規]をクリックします。

      Note

      スキャン ルール セットは、スコープ内のファイルの種類と分類ルールを決定します。カスタム ファイルの種類またはカスタム分類ルールをスキャンの一部として含める場合は、カスタム スキャン ルール セットを作成する必要があります。

    2. [接続元の種類]をAzure Data Lake Storage Gen2に変更して以下の値を適切なフィールドに入力し[続行]をクリックします。

      • スキャン ルール セット名 
        twitter_scan_rule_set
      • スキャン ルールの説明 
        Parquetファイルを検出し、Twitterのハンドルを分類するためのカスタム スキャン ルールセット。

    3. PARQUETを除く全てのファイルタイプの選択をクリアして[続行]をクリックします。

    4. 選択されているシステム ルールをクリアしtwitter_handleカスタム分類規則を選択し[続行]をクリックします。

    5. 除外パターンを入力せずに[作成]をクリックします。

      Note

      除外パターンは、スキャン中に除外する資産を Microsoft Purview に指示します。スキャン中に、Microsoft Purview は資産の URL をこれらの正規表現と比較します。上記の正規表現のいずれかに一致するすべてのアセットは、スキャン中に無視されます。

    ADLS Gen2アカウントへのデータ アップロード

    1. twitter_handles.parquetをダウンロードします。 Azure Storage ExplorerでAzure Data Lake Storage Gen2アカウントに移動し[Blob Containers]を展開してrawコンテナーを開きます。（存在しない場合は作成します。）

    2. [新しいフォルダー]ボタンをクリックしフォルダ名にTwitterと入力して[OK]をクリックします。

    3. 新しく作成したフォルダを右クリックし[開く]をクリックします。

    4. [アップロード]ボタンをクリックし[ファイルのアップロード]を選択します。

    5. ダウンロードしたtwitter_handles.parquetを選択し[アップロード]をクリックします。

    ADLS Gen2アカウントのスキャン

    1. [データマップ]＞[ソース]に移動しAzure Data Lake Storage Gen2タイル内の[新しいスキャン]をクリックします。

    2. [接続のテスト]をクリックして資格情報にアクセスできることを確認し[続行]をクリックします。

    3. 規定ではAzure Data Lake Storage Gen2アカウントが選択されスコープ内のすべてのパスがスキャン対象となります。スコープを縮小するためにルートの選択を解除しTwitterフォルダのみを選択し[続行]をクリックします。

    4. カスタムスキャン ルール セットのスコープを検証するには[詳細の表示]をクリックします。

    5. カスタム スキャン ルール セットにPARQUETファイル タイプとカスタム分類規則twitter_handle が含まれていることを確認し[OK]をクリックします。

    6. カスタム スキャン ルール セットtwitter_scan_rule_set を選択し[続行]をクリックします。

    7. スキャン トリガーを[一度だけ]に設定し[続行]をクリックします。

    8. [保存して実行]をクリックします。

    9. スキャンの進行状況を表示するには[ソース]に移動し[Azure Data Lake Storage Gen2]タイルの[詳細の表示]をクリックします。

    10. [最新の情報に更新]を定期的にクリックしスキャンの状態が[完了]になるまで更新します。所要時間は約5～10分です。

    分類による検索

    1. スキャンが完了したら検索バーにアスタリスク（*）を入力しワイルドカード検索を実行します。

    2. フィルタ パネル内の[分類]を[Twitter Handle]に設定して検索結果を絞り込みます。アセットの詳細を表示するにはtwitter_handles.parquetをクリックします。

    3. [概要]タブでスキーマ分類にTwitter Handleが含まれていることを確認します。分類された列を確認するには[スキーマ]タブに移動します。

    4. [スキーマ]タブではAccount Nameが分類された列であることが確認できます。
